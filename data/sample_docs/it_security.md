# (예시) 사내 IT 보안 가이드 — RAG 실습용 샘플 문서

> 실습 6(RAG)용 **가상의 회사 정책 문서**입니다. 실제 규정이 아닙니다.

## 계정 / 비밀번호
- 사내 비밀번호는 **최소 12자**, 영문 대소문자·숫자·특수문자를 포함해야 합니다.
- 비밀번호는 **90일마다** 변경하며, 최근 3개와 동일하게 설정할 수 없습니다.
- 모든 핵심 시스템은 **2단계 인증(MFA)** 이 필수입니다.

## API 키 / 비밀 관리
- OpenAI 등 외부 API 키는 **코드에 직접 작성(하드코딩) 금지**입니다.
- 키는 `.env` 또는 사내 **Vault** 에 보관하며, `.env` 파일은 절대 git 에 커밋하지 않습니다.
- 키가 노출된 경우 즉시 폐기(rotate)하고 보안팀에 신고합니다.

## 외부 LLM 사용 규정
- 고객 개인정보·미공개 재무정보는 외부 LLM API 에 입력할 수 없습니다.
- 사내 문서를 RAG 에 사용할 때는 **사내 전용 임베딩/저장소**를 사용합니다.
- 외부 서비스에 보낸 데이터는 캐시·학습에 사용될 수 있으므로 민감정보는 마스킹합니다.
