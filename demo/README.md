# 완성형 데모 — OpenWebUI + 로컬 LLM (CPU만, 5분 시연)

과정 **시작 전**, "3일 뒤 속까지 이해하게 될 AI 챗봇 서비스"의 **완성형**을 먼저 띄워 봅니다.
ChatGPT 같은 화면(대화 기록·모델 선택·파일 업로드)이 **이미 다 들어있는** 오픈소스 UI(OpenWebUI)를,
**내 PC 안에서 도는 경량 LLM(Ollama + qwen2.5:1.5b)** 에 연결해 단 몇 줄로 실행합니다.

> 메시지: **띄우는 것(도입)은 이렇게 쉽다.**
> 하지만 내 문서 연결(RAG)·비용·환각·보안에 손대는 순간 **안의 원리**가 필요하다 → 그게 이 과정.

**왜 OpenAI 가 아니라 로컬 모델?** 키 발급·과금·네트워크/방화벽 이슈가 전부 사라지고,
"외부 API 호출이 아니라 **내 노트북 안에서 LLM 이 직접 돈다**"를 눈으로 보여줄 수 있어 데모에 더 강력합니다.

## 사전 준비
- Docker Desktop (이미 과정 준비물)
- (인터넷은 **최초 모델 다운로드 때만** 필요 — 이후엔 오프라인 동작)

## 실행 (2줄)
```bash
cd handson/demo
docker compose up -d        # 이미지 pull + 모델(qwen2.5:1.5b ≈0.9GB) 자동 다운로드
```
→ 최초 1회만 모델 받는 시간(수 분, 회선에 따라 다름)이 걸립니다.
→ 받고 나면 브라우저에서 **http://localhost:3000** 접속 → 바로 대화 시작.

> 진행 상황 보기: `docker compose logs -f ollama-pull` (모델 다운로드 로그)

### 모델 선택
화면 좌상단 모델 드롭다운에서 **qwen2.5:1.5b** 선택(자동 등록됨).
- 더 가볍게: `docker-compose.yml` 의 `ollama pull qwen2.5:1.5b` → `qwen2.5:0.5b`
- 품질 우선: `gemma2:2b` (속도는 약간 느려짐)
- 추가 모델은 UI 의 *Settings → Models* 에서 모델명 입력으로 직접 pull 가능.

## 시연 시나리오 (5분)
1. 일반 질문 1개 → 답변이 **CPU에서 스트리밍**으로 또르륵 나오는 것 보여주기
2. "지금 이 답변은 인터넷 너머가 아니라 **이 노트북 안**에서 나왔다" 한마디
3. (선택) 문서 업로드 → "이게 RAG입니다. Day 3 에 직접 만듭니다"
4. 마무리: *"이 박스 안에서 무슨 일이 일어나는지, 3일간 직접 열어봅니다."*

## 정리
```bash
docker compose down          # 컨테이너 중지·제거 (대화 기록·모델은 볼륨에 보존)
docker compose down -v       # 볼륨까지 삭제 (모델 가중치 포함 완전 초기화)
```

## 자주 묻는 것
- **첫 응답이 느려요.** 최초엔 모델 로딩 시간이 듭니다. 한 번 돌린 뒤(워밍업) 시연하세요.
  CPU 사양이 낮으면 `qwen2.5:0.5b` 로 낮추면 훨씬 빨라집니다.
- **로그인이 없네요?** 데모 속도를 위해 `WEBUI_AUTH=False` 로 껐습니다.
  운영처럼 계정을 보려면 `WEBUI_AUTH=True` 로 바꾼 뒤 `docker compose down -v && docker compose up -d`.
- **포트 3000 이 이미 사용 중?** `docker-compose.yml` 의 `"3000:8080"` 을 `"3001:8080"` 등으로 변경.
- **모델 목록이 비어 있어요.** `ollama-pull` 다운로드가 끝나기 전입니다 —
  `docker compose logs -f ollama-pull` 로 "모델 준비 완료" 확인 후 브라우저 새로고침.
